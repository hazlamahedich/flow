## What Went Well

1. **Zero production incidents** — Across a complex 4-story epic (new data model, persistent timer with atomic RPC, AI agent, edit flow with invoice seam), zero user-facing failures on ship.
2. **`stop_timer` Postgres RPC** — Correctly identified that Supabase JS client cannot wrap DELETE + INSERT in a transaction; implemented a `SECURITY DEFINER` RPC for guaranteed atomicity. Right call under constraint.
3. **Pre-dev adversarial review on Story 5-3** — The team ran adversarial review before implementation. Discovered 2 of 4 original ACs were untestable without invoice tables. The `InvoiceEditGuard` seam pattern emerged from that session. Zero wasted implementation work. This should become standard practice.
4. **5-4 anomaly detection unit tests** — 22+ tests covering gap, overlap, and low-hours detection before review pass. NFR02 performance test (500 entries, 90 days, <30s) is regression protection.
5. **`signal_key` computed column workaround** — PostgreSQL B-tree indexes cannot cover `uuid[]` columns. Team discovered this at implementation time and invented a computed `signal_key text` column as the uniqueness carrier. Documented in project-context.md.
6. **T1-T6 from Epic 4 retro fully applied** — PgBoss DI (boss-di.ts), SQL CTE scope creep fix, RLS client_inbox_id filter, UX timer+brief spec, Epic 5 ATDD scaffolds, story template Pre-Dev Dependency Scan section — all verified applied.

## What Didn't Go Well

1. **RLS security bugs caught in review — third consecutive epic** — Story 5-1 CRITICAL findings: stale predicate `wm.removed_at IS NULL` (should be `wm.status = 'active'`), and INSERT policy missing `user_id = auth.uid()` AND `member_client_access` scope (any member could insert entries for clients they had no access to). Epic 4 retro said security discipline belongs in implementation — did not happen.
2. **A1-REVISED deferred cap gate not enforced** — Story 5-1 Round 1 had 10+ deferred items (exceeds the 5-item cap). No record of named Architect+PM exception approval. Gate exists as a rule; enforcement mechanism does not exist in tooling.
3. **5-4 gap/overlap detection non-functional for MVP** — `time_entries` schema has no `start_time`/`end_time` columns. Two of three anomaly detection types (gap, overlap) return empty arrays for all production entries. FR49 partially unmet. Story 5.4a is now mandatory before Epic 7.
4. **RLS isolation tests skipped in 5-1** — Explicitly flagged in review as "add before shipping to production." These prove the patched RLS policies actually work. Missing as of epic close.
5. **AC0 (test-first) not auditably enforced** — Tests exist in all stories. Whether tests were written red-before-green cannot be verified from story files — no commit SHA record.
6. **Drizzle/migration schema drift** — Both 5-1 and 5-4 had Drizzle schemas missing constraints present in SQL migrations (UNIQUE, CHECK). Pattern: migrations correct, Drizzle schemas lag.

## Key Insights

1. **Pre-dev adversarial review is a multiplier** — 5-3 proved it. The cost of a pre-dev adversarial session is a fraction of discovering scope problems mid-review. Make it standard for any story with external dependencies or uncertain scope.
2. **Gate, not guideline — still the lesson from Epic 4** — A1-REVISED was documented. It did not fire because enforcement requires tooling, not memory. Script the deferred count check.
3. **Shipped ≠ production-hardened** — Zero incidents is real. But missing RLS isolation tests mean the security proof is absent. The code is probably right; we just can't demonstrate it.
4. **5.4a is now on the critical path** — The Time Integrity Agent is a low-hours detector at MVP. Gap/overlap detection is schema-blocked. Story 5.4a must exist and be sequenced before Epic 7.

## Significant Discovery

**Story 5.4a is a mandatory prerequisite for Epic 7 (Invoice Reconciliation).**

The Time Integrity Agent (5-4) ships with gap and overlap detection non-functional because `time_entries` has no `start_time`/`end_time` columns. Low-hours detection works. Gap and overlap detection returns empty arrays for all production entries.

Epic 7's invoice reconciliation depends on the Time Integrity Agent having complete anomaly data. Story 5.4a must:
- Add `start_time`/`end_time` to `time_entries` (migration)
- Add time picker to log-time modal and edit modal
- Wire gap/overlap detection in `anomaly-detection.ts`
- Be sequenced: Epic 6 (parallel) → 5.4a → Epic 7

**Epic 6 plan is unaffected.** Epic 7 plan must account for 5.4a.

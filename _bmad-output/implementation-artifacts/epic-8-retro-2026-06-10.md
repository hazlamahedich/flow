## What Went Well

1. **Story re-slicing (8-1 → 8-1a/8-1b/8-1c) was the best structural decision of the epic** — Each slice had clear scope boundaries and clean dependency chains. 8-1a was generation+persist, 8-1b was templates, 8-1c was versioning+regeneration. No ambiguity.

2. **Conditional-write versioning pattern in 8-1c** — `UPDATE ... WHERE version = $expected AND status = 'draft'` and `INSERT INTO ... SELECT ... WHERE status IN ('sent', 'viewed')` eliminated race conditions without read-then-write. Pure SQL doing what SQL does best.

3. **Edge case matrices consistently applied** — Epic 7's P3 action item (AC template upgrade) paid off. 8-1a had 6 edge cases, 8-1c had 10, 8-2 had 10, 8-3 had 10. Result: 90/90 ATDD tests passing at epic close.

4. **Deterministic agents proved more reliable than LLM agents** — Client Health (8-3) and Friday Feeling (8-4) had zero hallucination risk, simpler testing, and faster implementation than Weekly Report Agent (8-2). LLM should only be used when narrative generation is the explicit requirement.

5. **Adversarial pre-dev reviews caught critical issues before code** — 8-1c's `version_group_id` replacing linked-list traversal, 8-2's cross-tenant leak risk, monorepo violation, trust gate ordering. All caught in spec review, not code review.

6. **Close-out gate mostly maintained** — Zero stories closed with unresolved findings at the `[Review]` tag level. The gate held.

7. **Signal emission pattern is proven** — Client Health Agent (8-3) emits `client.score_changed` signals that Weekly Report Agent (8-2) can consume. Cross-agent communication via database records works.

8. **Three different AI agent models delivered working code** — kimi-k2.6, Antigravity (DeepMind), glm-5.1 all produced shippable stories. Resilience in the toolchain.

## What Didn't Go Well

1. **8-1a had 26+ review findings — highest density in project history** — Silent error swallowing on 4 aggregation queries, `String(null)` corruption, sequential queries that should've been `Promise.all`, zero success-path tests for a 277-line action. The aggregation action needed 27 patches in a single review cycle.

2. **8-2 had CRITICAL cross-tenant data leak risk** — `aggregateReportData` was in `apps/web` and didn't accept `workspaceId` as a parameter. When 8-2 needed it in `packages/agents/weekly-report`, the function either crashed on `cookies()` or ran service_role queries without workspace scoping. Caught by adversarial review, not testing.

3. **Shared helper extraction forced mid-epic** — Moving `aggregateReportData` from `apps/web` to `@flow/db/queries` was necessary but disrupted 8-2's timeline. The monorepo violation should've been caught during 8-1a architecture review.

4. **pgTAP RLS tests deferred in 3/6 stories** — 8-3, 8-4, and partially 8-1a had RLS tests deferred because Supabase wasn't running locally. RLS is the security perimeter. Unverified RLS policies are assumptions, not guarantees.

5. **8-3 marked done with unchecked green-phase tasks** — File header says `ready-for-dev` but sprint-status says `done`. Tasks 2.5, 2.6, 3.8, and Task 10 (full green-phase) are still unchecked. Close-out gate violation.

6. **Agent model quality varied significantly** — kimi-k2.6 produced 26+ findings in 8-1a. Antigravity had 20 findings in 8-2 (but more critical). glm-5.1 had 8 findings in 8-4. The spread suggests model capability matters for code quality.

7. **Pre-existing test failures left unaddressed** — 8-4 dev notes reference dashboard test failures, calendar/inbox typecheck errors, and @flow/ui lint errors as "unrelated to this story." But they compound across epics.

## Key Insights

1. **Story re-slicing is a superpower** — 8-1 → 8-1a/8-1b/8-1c was the best structural call of the epic. Each slice was focused, shippable, and had clean dependencies. Apply this pattern to any story that feels "too big" during refinement.

2. **service_role without workspaceId is the #1 systemic risk** — 5 agent modules use service_role (inbox, calendar, time-integrity, weekly-report, client-health, friday-feeling). Every query must be audited before building external-facing features (Epic 9 portal).

3. **Deterministic > LLM for agents** — When the requirement is computation, not narrative, deterministic agents are simpler, faster, more testable, and zero hallucination risk. Use LLM only when prose generation is the explicit requirement.

4. **Supabase local is non-negotiable** — Deferred RLS tests in 3/6 stories is unacceptable. Fix the environment (Docker mount issue), enforce the gate (pgTAP must pass before `done`).

5. **Conditional-write pattern should be standard** — 8-1c's SQL-level status+version check eliminated race conditions without optimistic locking libraries. Apply to all mutable shared-state operations.

6. **Shared helpers belong in packages, not apps** — The 8-2 monorepo violation (importing from apps/web into packages/agents) should be a pre-dev checklist item. If a feature in `apps/web` might be needed by `packages/agents`, build it in `@flow/db/queries` from the start.

7. **Agent model quality correlates with finding density** — Track which model built which story. Use data to inform model selection for future epics.

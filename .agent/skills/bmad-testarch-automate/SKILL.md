## Available Scripts

- **`scripts/resolve-customization.py`** -- Resolves customization from three-layer TOML merge (user > team > defaults). Outputs JSON.

## Resolve Customization

Resolve `inject` and `additional_resources` from customization:
Run: `python3 scripts/resolve-customization.py bmad-testarch-automate --key inject --key additional_resources`
Use the JSON output as resolved values.

1. **Inject before** -- If `inject.before` resolved to a non-empty value, prepend it to your active instructions and follow it.
2. **Available resources** -- Note the `additional_resources` list. Do not read these files now; they are available for the injected prompt or workflow steps to reference when needed.

Follow the instructions in [workflow.md](workflow.md).

## Post-Workflow Customization

After the workflow completes, resolve `inject.after` from customization:
Run: `python3 scripts/resolve-customization.py bmad-testarch-automate --key inject.after`

If resolved `inject.after` is not empty, append it to your active instructions and follow it.

## Phoenix / Ecto Scaffold Alignment Pitfalls

When activating ATDD scaffolds on a Phoenix/Ecto project, these checks prevent green-but-wrong suites:

1. **Do not trust scaffold table lists.** Query actual schema state:
   ```
   SELECT table_name FROM information_schema.tables WHERE table_schema = 'public'
   SELECT indexname FROM pg_indexes WHERE schemaname = 'public'
   ```
   Use results to update `@required_tables` / `@expected_indexes` in data-model tests. Remove phantom tables (`sequence_steps`, `webhooks`, `source_configs`) if migrations have diverged.

2. **Ecto SQL rows are list-of-lists, not list-of-tuples.**
   ```elixir
   # WRONG (commonly generated by LLM)
   [{default, nullable}] = result.rows
   # RIGHT
   [[default, nullable]] = result.rows
   ```

3. **RLS policies are silently bypassed by PostgreSQL superusers.** If the test DB role (`postgres` by default) is a superuser, cross-tenant visibility assertions will fail even when the code under test is correct.
   - Use `@moduletag :skip` for entire RLS-focused modules (`repo_rls_test.exs`, `row_level_security_test.exs`).
   - Use `@tag :skip` for isolated RLS-dependent assertions inside otherwise-runnable modules, with an in-file NOTE comment describing the blocker.
   - Resolution requires a non-superuser test role in `config/test.exs`, not a code change.

4. **Verify real modules exist before deleting stubs.** Only remove `test/support/atdd_stubs.ex` after confirming the real context module (`lib/{app}/{context}.ex`) defines the expected functions.

5. **Skip unimplemented UI conditionals with a NOTE.** If a controller does not assign `current_workspace`, a layout template that conditionally renders a credit badge will not show. Skip the assertion, leave a NOTE referencing the missing plug/assignment.

See `references/phoenix-scaffold-alignment.md` for full recipes and copy-paste SQL.
